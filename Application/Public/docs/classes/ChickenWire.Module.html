    Module class containing information on loaded modules

    A module is a seperate set of models, views and controllers contained within your application. Modules have the same structure as an Application, with the exception of the settings for how the module behaves inside the Application. These settings can either be placed inside a Module.php file in the module's root directory, for example:

    Module.php

     $module->namespace = "SomeModule";

    This file will be automatically loaded when the module is loaded. You can optionally also configure the module inline, through the load() method, like:

    Module::load("SomeModule");      // No extra configuration
Module::load("SomeModule", array(
    "namespace" => "SomeModulesNamespace"       // Overriding default settings
));

    When neither of the configuration options is used, the values will be guessed as follows:

    Setting Guess Example for SomeModule Description
    namespace The name of the Module SomeModule The PHP namespace for your module.
    path MODULE_PATH + name of the module /srv/www/htdocs/Modules/SomeModule The full root path of the module.
    urlPrefix / + slugified name of the module /somemodule The prefix for all routes defined in this module. You can also enter an empty string, so the routes will be the same as the application.

    When you use autoLoadModules (see Application), all subdirectories in the Modules/ directory will be automatically loaded, with no extra configuration (except the Module.php config file). It will assume the name is the same as the directory name.

    see \ChickenWire\Application
